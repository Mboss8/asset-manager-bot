# 角色权限稳定规则

## 当前策略

- OWNER：全局最高权限。
- ADMIN：系统、成员、设置、BI 推送。
- PM：项目、任务、需求评审。
- FINANCE：财务收支、审核、报表。
- MEMBER：提交任务、需求、文档、报销，查看基础业务数据。
- GUEST：只读公开看板，不允许进入项目/任务/需求/文档/财务业务模块。

## 关键策略

- `ALL_USERS` 包含 GUEST，只用于主页、BI 公开看板等入口。
- `MEMBER_OR_ABOVE` 不包含 GUEST，用于业务模块和写操作。
- 黑名单用户由 bot 全局 middleware 拦截，不能通过旧按钮绕过。

## 验证路径

- GUEST：/menu 只应看到数据看板。
- MEMBER：可以进入任务、需求、文档、报销；不能进入成员/设置。
- FINANCE：可以进入财务审核、月报。
- PM：可以管理项目、任务、需求评审。
- ADMIN/OWNER：可以进入成员/权限、系统设置。
- 黑名单用户：任何按钮/命令都会被拦截。
